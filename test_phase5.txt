### Schema normalizer behavior (fetch-market-data.ts)
Backend payload (current orange-news-automation schema):
  assetClass: "index" | "crypto" | "commodity" | "forex"  (frontend reads as `asset`)
  history1w: [{date, close}, ...]                         (frontend reads as number[])

Normalizer (normalizeBackendInstrument):
  - Maps assetClass → asset (with "forex" → "fx" translation)
  - Flattens history1w/1m: extracts .close from each point
  - Idempotent: passes through canonical shape unchanged
  - Returns null if required fields missing (logged via mock fallback)
